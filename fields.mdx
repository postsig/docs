## Contract Metadata

| Name | Field ID | Description |
| --- | --- | --- |
| Vendor | `vendor` | Vendor name from global vendors stable
| Contract Type | `type_id` | Contract type |
| Contract Status | `status` | Either **Active**, **Inactive**, or **Unconfirmed**
| Will Not Renew | `will_not_renew` | Users can mark contracts for non-renewal and they will archive at the end date
| All Parties Signed | `all_parties_signed` | Indicates whether the contract has been executed by all parties. Displays "Not Executed" badge when value is "No" |
| Renewal Type | `renewal_type` | **Auto**, **Manual**, or **One-Time** |
| Auto Renewal | `auto_renewal` | **Replaced by renewal_type** Y/N, if a contract contains a stipulation for renewing automatically at the end of the initial term |
| Asset Classes | `asset_classes` | Financial product categories tagged based on contract content. [See full taxonomy →](#asset-classes) |
| Tags | `tags` | User-created tags applied to contracts |
| Summary | `summary` | An AI generated summary of the spirit of the agreement that was entered into by the client and vendor |
| PostSig Notes | `postsig_notes` | Notes from PostSig about a contract |

## Date Fields

| Name | Field ID | Description |
| --- | --- | --- |
| Start Date | `term_start_date` | The calendar date on which the contract begins |
| End Date | `term_end_date` | The calendar date on which the contract term ends |
| Cancellation Date Range | `cancel_by_date` | The date range that determines the date by which a contract needs to be cancelled to avoid automatic renewal, expressed in days |
| Cancel By Date | `cancel_date` | The date by which a contract must be cancelled to avoid auto renewal |
| Execution Date | `execution_date` | The date the contract was signed. In the event that parties signed on different dates, the latter will be identified as the execution date |
| Subscription Term | `subscription_term` | The length of the contract term, expressed as months |
| Multi-Year Agreement | `multi_year` | If a contract is for more than one year; contracts with auto renewal clauses are considered multi year |

## Permissions & Scope of Use

| Name | Field ID | Description |
| --- | --- | --- |
| End Users | `end_users` | Detail about the number of users, group/department/job function, user locations, may contain tiered pricing information |

| Distribution Rights | `distribution_rights` | The client's rights and obligations as it relates to distribution of the product sold by the vendor |
| Geographic Restrictions | `geo_restrictions` | The geographic restrictions of the data or product sold by the vendor |
| Derivative Works | `derivative_works` | Addresses if a user can combine the vendor data into other offerings to be shared with clients or used internally |

## NDA-Specific Fields

| Name | Field ID | Description |
| --- | --- | --- |
| Extended Confidentiality Term | `extended_confidentiality_period` | NDA-specific date field |
| Purpose | `purpose` | Describes the relationship between the parties and what the contract is meant to oblige the counterparties to during the course of the relationship |
| Mutual NDA | `mutual_nda` | Identifies the NDA as being unilateral or bilateral |
| Confidential Information | `confidential_information` | Describes the general and specific nature of the information to be shared between the counterparties to the agreement |
| Permitted Use | `permitted_use` | Describes what either party can and cannot do with regard to the confidential information |
| Term/Termination | `term_termination` | Describes the length of the agreement and any means by which the agreement can be effectively ended by either party |
| Non-Use/Non-Disclosure | `non_use_non_disclosure` | Describes how the receiving party is obliged not to disclose or use confidential information for anything other than the use cases prescribed in the NDA |
| Maintenance of Confidentiality | `maintenance_of_confidentiality` | Describes the obligations of the counterparties in regard to maintaining discretion while dealing with one another's confidential data and information |
| No Obligation | `no_obligation` | Describes specifics around what the agreement does not oblige either party to do during the course of the agreements |
| No License/Ownership | `no_license_ownership` | Describes details around persistence of property rights with regard to the confidential information that might be exchanged during the course of the agreement. Also defines property rights that arise from newly created IP |
| Exclusions/Exceptions | `exclusions_exceptions` | Describes carve-outs or information expressly not covered by the agreement |
| Non-Solicitation of Employees | `non_solicitation_of_employees` | Describes the extent to which the counterparties are obliged not to employ or retain one another's employees |
| Remedies | `remedies` | Describes the entitlements of either party should confidentiality be breached during the course of the agreement |
| No Warranty | `no_warranty` | Describes the warranties either party makes, or does not make, regarding the nature of the confidential information that may be shared |
| Disclosure Required By Law | `disclosure_required_by_law` | Describes what transpires when either party is legally compelled to disclose anything related to, and including, the confidential information |
| Miscellaneous | `miscellaneous` | Captures any detail with regard to the agreement that isn't effectively covered in other sections of the contract |

### NDA Risk Indicators

Automated risk assessment for NDA contracts. Contains 7 specific risk indicators.

<Note>
Risk levels are calculated as: **High Risk** (4+ flags), **Medium Risk** (1-3 flags), **Low Risk** (0 flags)
</Note>

| Risk Indicator | Field ID | Description |
| --- | --- | --- |
| Perpetual NDA | `perpetual_nda` | This is ongoing with no end of term date specified |
| Unilateral NDA | `unilateral_nda` | This will obligate ONLY the signing party to confidentiality |
| Non-Solicitation | `non_solicitation` | This obligates one or both parties to not hire or attempt to hire an employee of the counterparty |
| Uncapped Liability | `uncapped_liability` | This has no upward limit on monetary damages or other liabilities that can be incurred by one or both parties |
| Foreign Jurisdiction | `foreign_jurisdiction` | This stipulates any controversies arising will be adjudicated outside of the United States |
| No Carve-Out Provisions | `no_carve_out_provisions` | This fails to stipulate what doesn't constitute confidential information (i.e. publicly known information, compelled disclosure by courts) |
| Post End-of-Term Obligations | `post_end_of_term_obligations` | This has a term beyond the end of term that obliges continued confidentiality to one or both parties |
